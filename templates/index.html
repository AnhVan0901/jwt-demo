JWT Security Demo

Đề tài tập trung nghiên cứu cơ chế xác thực và phân quyền người dùng trong hệ thống API thông qua JSON Web Token (JWT). Thông qua việc xây dựng API mẫu, đề tài phân tích các rủi ro bảo mật khi JWT được cấu hình không an toàn, từ đó mô phỏng các kịch bản tấn công và đề xuất giải pháp phòng chống.

JWT được sử dụng rộng rãi trong các hệ thống web và ứng dụng hiện đại, tuy nhiên nếu lập trình sai hoặc cấu hình yếu, JWT có thể trở thành điểm yếu nghiêm trọng dẫn đến leo thang đặc quyền.

Backend & Security Tools

Chuẩn bị môi trường phát triển backend sử dụng NodeJS hoặc Python Flask để xây dựng API. API bao gồm chức năng đăng nhập và phân quyền dựa trên JWT. JWT được cấu hình sử dụng thuật toán HS256 với secret yếu nhằm phục vụ mục đích demo tấn công.

  • NodeJS hoặc Python Flask
  • JWT (HS256)
  • Postman / curl để gửi request
  • Công cụ jwt.io để phân tích token

Tiến hành demo tấn công JWT

Tấn công 1 – Weak Secret (HS256)

Demo 1

JWT Payload Manipulation

Trong kịch bản này, API sử dụng secret yếu để ký JWT và không kiểm tra chặt chẽ tính toàn vẹn của chữ ký. Người tấn công trước tiên đăng nhập bằng tài khoản user thông thường để lấy JWT hợp lệ.

Sau đó, JWT được dán vào công cụ jwt.io để giải mã. Kẻ tấn công chỉnh sửa payload, thay đổi trường role từ user sang admin, sau đó ký lại token bằng secret đã đoán được.

Token giả mạo này được sử dụng để gửi request đến API admin. Do hệ thống không xác minh chữ ký đúng cách, API chấp nhận token và cho phép truy cập trái phép.

Tấn công 2 – None Algorithm Attack (Mô phỏng)

Demo 2

JWT Algorithm Confusion

Ở kịch bản tấn công này, hệ thống không giới hạn thuật toán JWT hoặc cho phép thuật toán none. Người tấn công chỉnh sửa header của JWT, đặt thuật toán ký thành none.

Payload tiếp tục được chỉnh sửa để thay đổi quyền từ user sang admin. Do JWT không được ký và API không kiểm tra thuật toán, token không chữ ký vẫn được coi là hợp lệ.

Kết quả là kẻ tấn công có thể truy cập các endpoint yêu cầu quyền admin mà không cần biết secret, gây ra lỗ hổng bảo mật nghiêm trọng.
